# Launch Copy Templates

Use these templates when announcing the project publicly. The goal is to be explicit that Relaydex is an independent fork of Remodex, not the official upstream app.

## GitHub release template

Title:

```text
Relaydex v0.1.0: Windows-hosted Codex control from Android
```

Body:

```text
Relaydex is an independent fork of Remodex focused on one workflow: controlling local Codex running on Windows from Android.

What this release adds:
- Android client support
- Windows-friendly local bridge launch for `codex app-server`
- QR pairing and raw pairing-payload paste flow

Important notes:
- This is not the official Remodex app.
- Credits to the original Remodex project by Emanuele Di Pietro.
- The Codex runtime still runs locally on your own computer. The phone is only a remote control client.

Repository:
<repo url>
```

## X post template (JP-ready structure)

Use this structure when writing the Japanese post:

```text
Line 1: Say Relaydex is an independent fork of Remodex.
Line 2: Say it lets you control local Codex running on Windows from Android.
Line 3: List the three additions: Android client, Windows bridge launch, QR or pairing-payload flow.
Line 4: State clearly that it is not the official Remodex app.
Line 5: Credit the original Remodex project by Emanuele Di Pietro.
Line 6: Link to the GitHub repository.
```

## Short anti-confusion checklist

- Say `independent fork of Remodex` in the first paragraph.
- Say `not the official Remodex app`.
- Link to the upstream repository.
- Do not imply endorsement by the upstream author or by OpenAI.
- Use your own app name, package name, screenshots, icon, and signing identity.
