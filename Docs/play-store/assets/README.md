## Phone screenshots

All screenshots in `phone-screenshots/` are portrait PNGs at `1440x3120`.
The curated set avoids screenshots that show error dialogs, failed relay states, or expired host status.

Suggested upload order:

1. `01-home-connected.png` — paired host overview and project/thread landing state
2. `02-thread-ready.png` — active conversation view with a clean completed exchange
3. `03-runtime-status.png` — composer state with quick runtime controls and no error or expired-state banners
4. `04-project-switching.png` — sidebar and project/thread switching
5. `05-command-autocomplete.png` — keyboard-first composer entry flow
6. `06-runtime-controls.png` — model/runtime configuration sheet
